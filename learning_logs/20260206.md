# SQL 기본
## ORDER BY
- 어떤 순서를 볼까
  1. 문자를 오름/내림 차순
  2. 숫자를 오름/내림 차순

- 예제문제 : users에서 id 기준으로 오름차순 정렬 해보기
```sql
SELECT * FROM users ORDER BY id ASC;
```
- ASC : 오름차순
- DESC : 내림차순

- city 기준으로 오름차순 정렬
```sql
SELECT * FROM users ORDER BY city ASC;
```

- created_at 기준으로 내림차순 정렬
```sql
SELECT * FROM users ORDER BY created_at DESC;
```

- ORDER BY는 컬럼명을 몰라도 정렬을 수행하는것이 가능 _몇번째 컬럼인지 숫자로 입력해도 가능_ 하다는점

- users 테이블의 첫번째 컬럼 기준으로 내림차순 정렬
```sql
SELECT * FROM users ORDER BY 1 DESC;
```

- answp : users 테이블에서 username, phone, city, country, id를 순서대로 보여줄수있도록 SELECT문을 쓰고, 첫번째 컬럼을 기준으로 오름차순 정렬하시오
```sql
SELECT username, phone, city, country, id
	FROM users ORDER BY 1 ASC;
```

- 만약 id를 기준으로 오름차순 되기를 바랬다면 결과값이 생각한대로 나오지않았음. ORDER BY 절의 실행 순서를 파악해야 할 필요가 있는데 _SELECT적의 실행 결과를 기준으로 정렬_ 하기때문에 이상의쿼리문을 기준으로는 1번 컬럼이 username이 되어, username varchar기준으로 오름차순 정렬되었다고 해석할수 있음.

- SELECT로 컬럼 순서를 바꿧지만 id 순서로 ASC하기 위해선 컬럼명을 명시하는것이 좋음.
```sql
SELECT username, phone, city, country, id FROM users ORDER BY id ASC;
```

- ORDER BY의 기준을 복수로 잡는것도 가능. 한개 이상의 정렬 기준을 잡을수 있음.
`ORDER BY 1번 걸럼 ASC/AESC, 2번 컬럼 ASC/DESC`
와 같은 방식으로 작성할수 있는데, 이상의 겨우 1번컬럼을 기준으로 먼저 정렬을 수행한 다음에2번 컬럼에서 재정렬을 수행함.

- users에서 city, id 컬럼만 출력하고, 거주도시 기준으로 내림차순, 회원 아이디 기준오름차순 정렬
```sql
SELECT city, id FROM users
	ORDER BY city DESC, id ASC;
```

- 이상의 ORDER BY 부분을 해석하면, 먼저 city 기준으로 내림차순 정렬한 뒤, 동일한 cuty 값 내에서는 id를 기준으로 오름차순 정렬하라는 의미가 됨. 그렇다면 ORDER BY 이후에 명시하는 컬럼명의 순서도 신경을 쓸 필요가 있다는 것을 확인할수 있음.

# 묶어서 계산하기
## 데이터를 계산하는 함수

- 엑설에서 합 / 평균 / 개수 세는 등의 작업을 할때 함수를 사용한적이 있는데, SQL에서도 마찬가지로 함수를 사용해 동일한 작업을 할수 있음
  - 합 / 평균 / 개수 세는등의 _집계함수_
  - 문자열을 잘라내거나 / 대소문자 변경등을 수행하는 _일반함수_
등이 존재함

- 그룹별로 _집계 함수를 적용하기 위해 나누는_ `GROUP BY`
  - 전체 데이터를 함수를 적용하지만, 그룹별로 수치를 도출하는 일도 자주있음. 그런경우 GROUP BY를 이용해 데이터를 그룹으로 묶은후 필요한 함수를 적용하면 됨.
  ```SQL
  SELECT COUNT(id) AS cnt FROM orderdetails;
  ```
  - 이상의 코드의 경우는 orderdetails의 전체 건수를 확인하는게 되겠지만 procuct_id를 기준으로 적용한다든지 등의 일부 컬럼을 가지고 세세한 개수를 파악하는 것은 쉽지않음
  - 하지만 GROUP BY를 적용했을땐 '국가별 회원수를 계산하기' 혹은 '일별 매출을 계산하기' 등으로 좀 더 세분화 해서 사용할수 있음.

- GROUP BY로 계산한 결과를 필터링하는 HAVING
  - GROUP BY를 사용하면 국가별 회원수를 추출하는것은 가능한데, 그중에서 마케팅 효율을 높이기 위해 국가별 회원수를 추출하고 그 중에서 회원수가 10며 이상인 국가만 재추출하는등 _집계함수로 계산한 결과중에서 조건에 맞는 데이터만 필터링_ 하는 경우도 존재함.
  - 그러면 일단 집계 함수가 적용된 이우헤 WHERE을 써야 한다는건데, 집계되기 전에 미리 생각하고 WHERE절을 설계하는것이 쉽지않기 때문에 _GROUP BY 집계 함수의 `결과값`을 필터링할때는 `HAVING`을 씀_

## 집계 함수
  1. COUNT : row 개수을 셀때 사용
  2. AVG : 평균을 계산할때 사용
  3. SUM : 숫자형 데이터 값을 더할때 사용
```sql
SELECT COUNT(*) FROM users;
```
이상의 쿼리문은 회원 테이블의 row개수를 추출함. `COUNT(*)`는 대상 테이블에 있는 전체 행의 개수를 세는 역할을 함.

- users 에서 존재한느 국가의 수를 카운팅
```sql
SELECT COUNT(country)
  FROM users;
```
- 전체 row의 개수는 77개 지만, 더미 데이터 기주느 1번 row가 country가 null이기 때문에 76으로 출력됨.

- 이상에서 알수있는것은 COUNT() 함수는 컬럼의 값이 null이 아닌 행의 개수를 return한다고 할수있음.
- 그런데 금일 수업중 도시 기준으로 내림차순 헀을때 Seoul이 여러개 여쓰니까, Korea도 여러개니까 중복을 포함하여 카운팅이 되었다고 생각할수 있음.

- 중복제거 명형어 `DISTINCT`

- 예시 : users에서 존재하는 국가를 세기(중복제외)
```sql
SELECT COUNT(DISTINCT country) AS countries FROM users;

SELECT DISTINCT country AS countries FROM users;
```
![COUNT 미적용 DISTINCT](image-1.png)

이상의 이미지를 확인했을때 18이라고 출력했지만 COUNT() 적용했을때 17이 나옴. null은 집계함수 상에서 카운팅이 되지않는거지 그냥 보여 줄땐 그대로 나옴. 그래서 COUNT() 적용결과는 17이고, 미적용시에 18번 row까지 존재함.

- MIN() : 최저값 추출하는 함수 / 문자열에도 사용 가능.
```sql
SELECT MIN(컬럼) FROM 테이블;
```
- MAX() : 최대값 추출하는 함수 / 문자열에도 사용 가능.
```sql
SELECT MAX(컬럼) FROM 테이블;
```

  1. SUM
  2. AVG
  3. MAX

### 일반 함수(중 일부)
  1. ROUND(p1,p2,p3)
  2. SUBSTR(p1,p2,p3)
    - p2 : 시작위치
    - p3 : 가져올 문자열 개수
  3. LENGTH

- 집계함수는 여러 행의 데이터를 하나의 결과값으로 _집계_ 하는 반면, 일반 함수는 _한 행의 데이터에 하나의 결과값_ 을 대상으로 함. 따라서 **집계 함수는 SELECT 에서만 사용가능하지만, 일반 함수는 SELECT에서도 쓸수있고, WHERE절에서도 쓸수있음**

## GROUP BY
이상까지 수업내용에서 SELECT 문에 집계 함수를 적용하고, 일반 함수를 SELECT와 WHERE절에서 사용해봤음. 집계함수 SELECT절로 쓰게될 경우에 생겨나는 한계가 존재함. 데이터 전체가 아니라 원하는 기준으로 그룹을 나눠서 계산하기가 어렵다든지, 전체 회원수가 아니라 국가별 회원수를 나눈다든지 할때가 됨.
  - Korea는 몇 명인지, USA는 몇명인지 하려면 기존에는

  1. country가 Korea인거 카운트
  2. country가 USA인거 카운트
  한 다음에 카운트 해야함.

직접 해볼때.
1. 위의 과정을 거친 담에 답을 md파일에 작성하시오
```sql
SELECT COUNT(id) FROM users WHERE country = 'Korea';

SELECT COUNT(id) FROM users WHERE country = 'USA';

SELECT COUNT(country) FROM users WHERE country IN ('Korea', 'USA');
```
총 몇 명: 19

이상의 코드에서 생겨나는 문제가 뭐냐면 기본적으로 country내에 무슨 값이 있는지를 한번 체크해야 한다는 부분.

- 예시문제 : users에서 국가별 회원 수를 추출

```sql
SELECT country, COUNT(DISRINCT id)
  AS 국가별회원수
  FROM users GROUP BY country
```

이상의 퀄문을 실행헀을때 국가별 회원수를 확인할수있읍ㅁ.
GROUP BY는 _집계 함수와 함께_ 사용되며 GROUP BY 기준 컬럼은 SELECT 에서 집계 함수를 사용할때 묶어서 계산을 수행하는 기준이 됨.

- SELECT 절에만 집계함수를 쓸수있음.
- GROUP BY를 명시하지않았을때는 전체 row를 기준으로 집계 함수가 일어남 -> 테이블 전체를 기준으로 id 개수를 세든지 총합을 구하든지, 최대값, 최소값을 출력하든지 하게 됨.
- GROUP BY 컬럼명을 명시하게 될 경우에는 해당 컬럼의 중복되는 row값들을 다 합치게 됨. 그렇다면 GROUP BY까지 포함한 쿼리문의 연산 결과에 대한 전체 row의 개수를 미리 알수있음.
```sql
SELECT COUNT(DISTINCT country) FROM users;
```
- 거기에 GROUP BY 컬럼명a를 작성했다면 가기성을 위해 SELECT문에도 컬럼명 a를 명시해주는 것이 어떤 값에 집계함수 결과가 무엇인지를 명확하게 보여주는 방법이 될것.

- 이하는 GROUP이 이루어질 column을 명시 하지않았을때 가기성이 박살난 결과값
